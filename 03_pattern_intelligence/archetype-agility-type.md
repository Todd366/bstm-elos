# Archetype: Agility Type

**Definition:** A flexible or mobile business model (no fixed real estate) that trades overhead for a need for stronger coordination systems.

**Evidence:**
- Trial 003 — Nado Clothes Store (vehicle-based mobile retail)

**Diagnostic signature:** No permanent storefront; location changes during operating hours; transaction volume stays stable across moves *only if* inventory, location, and customer communication are actively synchronized.

**Required supporting systems:** real-time inventory tracking, live location/customer communication (WhatsApp/PWA), digital product showcasing as a storefront substitute.

**Known intersection:** Trial 007 (referenced in governance notes) revealed Agility-type infrastructure can carry Liquidity-type resource bottlenecks — see Archetype Evolution Rules below.

**Evolution state:** 🟢 Active — 1 confirming trial. Watch for a possible **Split** into "Dynamic Liquidity Type" if the Agility/Liquidity intersection repeats.
